# 不断学习

## 提交规范

推荐 commit-msg 规范遵守 [约定式提交](https://www.conventionalcommits.org/zh-hans/v1.0.0/)

提交类型:
-   feat: 新功能，而不是构建脚本的一个新功能
-   fix: 修复缺陷
-   refactor: 重构（即不是新增功能，也不是修改 bug 的代码变动）
-   perf: 优化或性能提升的改动
-   chore: 架构配置或工具的变动（即不是新增功能，也不是修改 bug 的代码变动）
-   docs: 文档的改动
-   style: 编码规范或代码风格的改动
-   test: 测试
-   build: 项目打包和构建脚本的改动
-   revert: 撤回修改，慎用
-   merge: 合并分支
